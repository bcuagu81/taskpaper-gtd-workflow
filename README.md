## 它到底是怎么工作的

用一句话解释：TaskPaper 把 GTD 方法论，翻译成了几个打字规则。

- 行末加冒号 `:`，这行就变成一个**项目**
- 行首加破折号 `-`，这行就变成一个**任务**
- 加 `@` 符号，后面跟任意文字，就是一个**标签**
- 普通文字行就是**备注**
- 用 Tab 键缩进，创建子任务

就这几条，再也没有别的了。但别小看这几条，配合过滤搜索，它能做到很多复杂工具才能做到的事。

比如你在搜索栏输入 `not @done`，立刻看到所有未完成任务；输入 `@due < 2026-04-01`，找出四月前截止的所有项目。这些搜索规则还可以保存到侧边栏，点一下就切换，类似 OmniFocus 的"透视"功能，但实现方式是纯文本查询。

## 几个让人真正爱上它的细节

**文件随便存、随便移**

因为数据就是一个文本文件，你可以把它扔进 iCloud Drive、Dropbox、OneDrive，任何你喜欢的网盘，同步到所有设备。不用依赖软件自己的同步服务，不用担心哪天软件倒闭了数据没了。

**第三方工具全兼容**

在 Mac 上用 nvALT、Sublime Text（装 PlainTasks 插件），在 iOS 上用 Editorial、TaskMator，都可以直接打开和编辑 `.taskpaper` 文件，数据互通，无缝配合。

**JavaScript 脚本，真的很有用**

这是 TaskPaper 比同类纯文本工具明显高出一截的地方。它内置了完整的 JavaScript API，你可以写脚本自动化各种操作——比如批量标记完成、自动整理项目、与其他 App 集成。官方社区还维护了一个 Extensions Wiki，里面有大量现成脚本可以直接拿来用，不懂写代码的用户也能受益。

**Dark Mode 支持 + CSS 主题定制**

支持暗色模式，也支持通过类似 CSS 的样式表自定义界面外观。字体、颜色、行高、背景色，都可以按自己的喜好来。

## 适合谁用

坦白说，TaskPaper 不是为所有人准备的。它的上手曲线比点点点那种 App 略高，但也真的不难，熟悉 Markdown 的人半小时就能完全掌握。

以下几类人会比较适合：

1. **习惯键盘操作的用户** — 基本不需要鼠标，全程键盘搞定
2. **反感被工具绑架的用户** — 数据是你的，格式是公开的，随时迁移
3. **有定制化工作流需求的用户** — 脚本 + 样式，可以深度打造自己的系统
4. **写作者、研究者** — 很多人用它做文章大纲、研究资料整理，因为纯文本+标签的组合管理碎片信息效果出奇的好

不太适合谁：需要团队协作、日历集成、甘特图的场景，TaskPaper 不擅长这个，用 OmniFocus 或 Notion 会更合适。
